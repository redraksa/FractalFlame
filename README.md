Высокопроизводительный генератор фрактального пламени на Java, использующий алгоритм Chaos Game и оптимизированный для работы в многопоточной среде.

В данной работе был реализован генератор фрактального пламени,
оптимизированный для работы в многопоточной среде.
Основной упор сделан на устранение «узких мест» стандартных подходов.

Алгоритм рендеринга
Программа использует алгоритм Chaos Game с применением нескольких
нелинейных вариаций и аффинных преобразований:

1. Реализовано 7 различных функций трансформации
2. Каждый пиксель аккумулирует цвет на основе весов примененных преобразований.
3. Реализована логарифмическая гамма-коррекция для выравнивания яркости и повышения детализации в темных областях фрактала.

Многопоточность и производительность
Для ускорения программы и избегания опасных ситуаций по типу гонки потоков
были применены следующие техники:

1. Вместо создания множества объектов-пикселей, используется массивы примитивов.
2. Каждый поток работает со своим локальным буфером
3. Рекомендуется запуск программы с флагом -Xmx4G и -XX:+UseParallelGC для минимизации пауз сборщика мусора при генерации точек.

Конфигурация и управление
Реализована следующая схема приоритетов:
Консольные аргументы > JSON-конфиг > значения по умолчанию

Пример запуска программы
Для генерации изображения с использованием файла конфигурации и переопределением количества потоков
mvn exec:java -Dexec.mainClass="academy.Application" -Dexec.args="--config=config.json --threads=16" -DargLine="-Xmx4G -XX:+UseParallelGC"

Проект включает в себя тесты производительности (RenderBenchmarkTest) для замера ускорения при масштабировании количества потоков.
